## Request Portal

The Request Portal screen lets people outside the maintenance team quickly submit issues without needing full access to the system.

### What this screen is for

The Request Portal is a simple public-facing form for reporting problems.  
Each submission becomes a work request that your team can review and handle.

### Who typically uses it

Tenants or occupants  
Customers or building staff  
Anyone allowed to report issues  

### Key things you can do

• Submit a new maintenance request  
• Provide contact details so the team can follow up  
• Describe the problem in plain language  
• Optionally attach a photo of the issue  

### Create a new request

1. Open the **Request Portal** link (usually `/request`).  
2. Enter your name and contact information.  
3. Choose or type the location of the problem.  
4. Describe what is wrong in clear, simple words.  
5. Attach a photo if the form allows it and it helps explain the issue.  
6. Click **Submit**.

### Layout overview

**Request form**  
Shows simple fields for your details, location, and description.  
Some portals may also allow you to attach a photo.

**Confirmation**  
After you submit, the page shows a short confirmation so you know the request was received.

### Typical workflow

Person submits a request in the Request Portal  
→ Request appears on the Work Requests screen  
→ Supervisor reviews and approves the request  
→ Work order is created and scheduled  
→ Technician completes the work  

### Tips

• Encourage requesters to include a clear description and photo when possible.  
• Share the Request Portal link with the right group of people (for example, tenants or site staff).  
• Use the Work Requests screen to triage and track everything that comes in through the portal.  

## Request Portal

### What this module is for

The Request Portal lets people submit maintenance requests without needing full access to the main application.  
It provides a simple way for occupants or customers to report issues.

### Who uses this module

Tenants or occupants  
Customers or location staff  
Anyone allowed to submit requests from outside the core team

### When you would use this module

- Reporting a new issue, such as a leak or equipment problem  
- Letting non-technical users send requests in a simple way  
- Feeding the Work Requests and Work Orders process

### Key things you can do here

- Open the portal at `/request` (or from shared links)  
- Fill out a request form with basic information  
- Submit the request so it can be reviewed and turned into work

### Main screen overview

When you open the **Request Portal** you see a focused page for submitting a request.  
The form prompts the user for key information such as name, contact details, location, and a description of the problem.  
The layout is designed to be simple so that first-time users can complete it without training.

### Step-by-step guide

#### Submit a maintenance request

1. Open the **Request Portal** page (usually `/request` or a shared URL).  
2. Enter your name and contact details.  
3. Provide the location of the issue (for example, property, building, or area).  
4. Describe the problem in clear, simple language.  
5. Attach a photo if the form allows it and it helps show the issue.  
6. Click **Submit** to send the request.

### Common workflows

Person submits a request in the portal  
↓  
Request appears in the Work Requests module  
↓  
Supervisor reviews and approves or converts to a work order  
↓  
Work order is scheduled and completed

### Tips and best practices

- Share the portal link widely with the people who should report issues.  
- Encourage requesters to include clear descriptions and photos.  
- Use the Work Requests module to triage and track what has been submitted.
