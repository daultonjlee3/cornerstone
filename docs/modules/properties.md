## Properties

The Properties screen lists the main sites or campuses you manage so you can organize work by place.

### What this screen is for

The Properties screen is the top level of your location structure.  
Each row is a property that can contain buildings, units, assets, and work.

### Who typically uses it

Operations managers  
Maintenance supervisors  
Asset managers  

### Key things you can do

• View all properties in your tenant  
• Open a property to see basic details  
• Use properties as filters in other modules such as Assets and Work Orders  

### Create a new property

1. From the left sidebar, go to **Properties**.  
2. Click the **New Property** or **Create** button if it is available.  
3. Enter a property name that matches what people use in the real world.  
4. Add any helpful notes such as city or site type.  
5. Click **Save**.

### Layout overview

**Header**  
Shows the page title and may include create and filter options.

**Property list**  
Each row is a property with a name and key details.  
Click a row to open more information.

### Typical workflow

Property created in Properties  
→ Buildings and units added under that property  
→ Assets and work orders linked to those locations  
→ Filters across the app can use property to narrow results  

### Tips

• Use names that match how your teams already talk about sites.  
• Keep properties up to date when sites are added, renamed, or retired.  
• Use properties to group reports and planning by campus or region.
