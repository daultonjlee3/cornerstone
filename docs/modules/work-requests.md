## Work Requests

The Work Requests screen shows all incoming maintenance requests so you can review, approve, and turn them into work orders.

### What this screen is for

The Work Requests screen acts like an inbox for new issues.  
Each row represents a request that someone submitted, with details such as who requested it, what is wrong, and where it is.

### Who typically uses it

Maintenance supervisors  
Operations managers  
Dispatchers  

### Key things you can do

• View all open and past maintenance requests  
• Open a request to see full details and any attached photo  
• Submit internal requests from inside the app  
• Decide which requests should become work orders  

### Create a new work request (internal)

1. In the left sidebar, go to **Work Requests**.  
2. Click the option or link to **Submit Request** (this opens `/requests/submit`).  
3. Enter your name and contact information.  
4. Choose the location and describe the problem in simple language.  
5. Attach a photo if it helps show the issue.  
6. Click **Submit**.

### Layout overview

**Header**  
Shows the title and a short description of the Work Requests list.

**Request list**  
Each row shows who submitted the request, where it is, and a short summary.  
Click a row to open detailed information about that request.

**Submit page (`/requests/submit`)**  
Provides a simple form for creating a new request from inside the app.

### Typical workflow

Person submits a request (from Request Portal or submit page)  
→ Request appears on the Work Requests list  
→ Supervisor reviews details and decides what to do  
→ Approved request is turned into a work order  
→ Work order is scheduled and completed  

### Tips

• Review new requests regularly so issues are not missed.  
• Use clear titles and descriptions when converting to work orders.  
• Close the loop by updating the requester through your normal communication channels.  

## Work Requests

### What this module is for

The Work Requests module collects and manages incoming maintenance requests.  
It acts as an inbox where supervisors can review, approve, reject, or convert requests into work orders.

### Who uses this module

Maintenance supervisors  
Operations managers  
Dispatchers

### When you would use this module

- Reviewing new requests from the Request Portal or other sources  
- Deciding which requests should become work orders  
- Tracking the status of requests over time  
- Communicating decisions back to requesters (where your process includes that step)

### Key things you can do here

- View a list of work requests at `/requests`  
- Open a request to see full details  
- Submit a new request internally from `/requests/submit`  
- Convert approved requests into work orders

### Main screen overview

When you open **Work Requests** from the sidebar you land on `/requests`.  
You see a table of requests with basic information such as who submitted the request, where it is, and what the issue is.  
The page header and any filters help you focus on new or outstanding items.

There is also a submit page at `/requests/submit` that provides an internal request form similar to the public Request Portal.

### Step-by-step guide

#### Review new work requests

1. Click **Work Requests** in the left sidebar.  
2. Look for new or unreviewed requests in the list.  
3. Click a request to open its details.  
4. Read the description, location, and contact information.  
5. Decide whether to approve, reject, or hold the request based on your process.  
6. Use the available actions to move the request forward, such as converting it into a work order.

#### Submit an internal request

1. Navigate to `/requests/submit` from the Work Requests area.  
2. Fill in the request form with your name, contact details, location, and description.  
3. Attach a photo if the form allows it and it would help.  
4. Submit the form to add the request to the Work Requests list.

### Common workflows

Request Portal or internal form  
↓  
Work Requests list  
↓  
Supervisor reviews and approves  
↓  
Work order created and scheduled  
↓  
Work completed and tracked in reports

### Tips and best practices

- Review incoming requests frequently so issues are not missed.  
- Use clear titles and descriptions when converting to work orders.  
- Keep request statuses up to date so you can see which items are still waiting for a decision.
